# Open-Source Boundary

The public KeyVeil repository is intentionally a generic reference.

## Included

- Policy and session data models.
- Approval verification and budget-store interfaces.
- A process-local reference implementation.
- Synthetic addresses, intents, scenarios, and receipts.
- Tests, CI, Docker packaging, and the reference workbench.

## Excluded

- Production trading or payment executors.
- Proprietary risk, routing, alpha, or wallet-profiling logic.
- Real customer, wallet, ledger, incident, or telemetry data.
- Credentials, provider inventories, private endpoints, and signing material.
- Internal thresholds, models, labels, and operational runbooks.

The repository history is created from a sanitized source tree. It does not
inherit commit history from private production repositories.
